# AgentDesk UX 리서치 보고서 v1.0

**작성일**: 2026년 2월 18일
**작성자**: ux-researcher
**버전**: v1.0
**분류**: 내부용 (Design Team)
**연관 문서**: `marketing/research/2026-Q1_target-customer-analysis.md`

---

## 리서치 개요

- **목적**: Phase 1 시장 조사 결과를 기반으로 AgentDesk 제품 설계에 활용할 페르소나, 여정 맵, 핵심 시나리오, 정보 아키텍처(IA) 초안, UX 원칙을 도출한다.
- **방법론**: 2차 데이터 분석 (시장 리서치 보고서, 경쟁사 UX 패턴, 업계 벤치마크), 합리적 가정 기반 페르소나 설계 (1차 인터뷰 전 가설 페르소나)
- **대상**: ICP-A (스타트업 CTO), ICP-B (프리랜서/1인 사업자), ICP-C (중소기업 비기술 부서장)
- **기간**: 2026년 2월 (Phase 1 완료 시점)
- **주요 한계**: 본 보고서의 페르소나와 여정 맵은 1차 사용자 인터뷰 없이 작성된 가설 기반 초안이다. 제품 출시 전 최소 5명 이상의 대표 사용자 인터뷰로 검증이 필요하다.

## 1. 사용자 페르소나 (User Persona)

### Persona 1: "빠른 결정자" 김성준 (ICP-A: 스타트업 CTO)

```
[기본 정보]
이름:     김성준
나이:     34세
직책:     CTO / 공동창업자
회사:     핀테크 스타트업 (직원 18명, 시리즈 A 준비 중)
위치:     서울 성수동
학력:     컴퓨터공학 학사, KAIST 출신
연봉:     8,000만원 + 스톡옵션

[기술 숙련도]
언어/툴:  Python, TypeScript, AWS, GitHub, Notion, Slack, Linear
AI 활용:  ChatGPT Pro, Cursor, GitHub Copilot 일상 사용
숙련도:   ★★★★★ (전문 개발자 수준, 새 도구 습득 빠름)

[일상 (하루 24시간)]
09:00   Slack 확인, 이슈 트리아지
10:00   개발팀 스탠드업 (30분)
10:30   코드 리뷰, PR 머지
12:00   점심 & 투자자 미팅 준비
14:00   Product 회의 (CEO, PM과 우선순위 조율)
16:00   핵심 기능 직접 코딩 or 아키텍처 리뷰
18:00   팀원 1on1 미팅
19:00   문서 작성 (API 스펙, 기술 설계서)
21:00   개인 공부, 사이드 프로젝트, X(트위터) 기술 커뮤니티

[목표]
- 시리즈 A 투자 유치 전 제품 안정성과 개발 속도 동시 달성
- 팀 규모 확장 없이 생산성 2배 향상
- AI를 통해 반복 업무(문서 작성, 코드 리뷰 초안, 슬랙 요약)를 자동화하고 싶음

[동기]
- "우리 팀이 쓰는 방식 그대로" 맞춤화할 수 있는 AI 도구
- 데이터가 쌓일수록 더 스마트해지는 시스템
- 영업/마케팅/CS 등 비개발 업무를 AI에 위임하고 싶은 욕구

[좌절감 (Pain Points)]
- ChatGPT와 사내 Notion, GitHub, Slack이 연결되지 않아 매번 컨텍스트를 복붙해야 함
- AI 에이전트를 직접 만들 수 있지만 시간이 없어 못 함 → "직접 짜면 3주, 사도 안 맞으면 돈 낭비"
- 여러 AI 구독료(Copilot, ChatGPT, Perplexity 등)가 월 $150 이상 → 단일 플랫폼으로 통합하고 싶음
- AI 출력 품질이 들쑥날쑥 → 팀에 배포하기 어려움
- LLM API 비용이 예측 불가 → 월말 청구서가 두려움

[사용자 인용문 (가상)]
"ChatGPT 창을 14개 열어 놓고 일하는 게 현실이에요.
 Notion에 있는 우리 도메인 지식이랑 GitHub 이슈가 연결돼서
 자동으로 답 줬으면 좋겠는데... 그런 게 없더라고요."

[사용 시나리오]
시나리오: 월요일 아침, 신규 팀원 온보딩
- 성준은 이번 주 백엔드 개발자 1명이 합류하는 상황
- 기존에는 Notion 문서 링크 10개를 정리해서 보내주고 반나절을 소비
- AgentDesk를 통해 "온보딩 에이전트"가 기존 문서, 코드베이스, 히스토리를
  자동으로 요약하여 신규 입사자에게 전달
- 성준은 핵심 코드 아키텍처 설명 30분만 투자하고 나머지는 에이전트가 처리
```

### Persona 2: "혼자서 다 하는" 이지은 (ICP-B: 프리랜서/1인 사업자)

```
[기본 정보]
이름:     이지은
나이:     29세
직업:     독립 콘텐츠 마케터 겸 카피라이터
형태:     프리랜서 (프리랜서 3년차, 법인 설립 준비 중)
위치:     서울 마포구 (홈오피스)
주요 클라이언트: IT 스타트업, 뷰티 브랜드, 교육 플랫폼 3~5개 동시 진행
월 매출:  약 700만원

[기술 숙련도]
툴:       Notion, Figma (기초), Google Workspace, 카카오워크
AI 활용:  ChatGPT, Claude, 뤼튼 간헐적 사용
숙련도:   ★★★☆☆ (파워유저이지만 개발/API 지식 없음)

[일상 (하루 24시간)]
08:30   커피 한 잔, 클라이언트 카카오톡/이메일 확인
09:00   그날의 납품 목록 정리 (Notion)
10:00   콘텐츠 제작 집중 타임 (SNS 카피, 블로그, 뉴스레터)
12:30   점심 + 유튜브 (AI 툴 사용법 영상 시청)
14:00   클라이언트 미팅/피드백 반영 작업
16:00   새 프로젝트 제안서 작성, 견적서 발행
18:00   SNS 포스팅 예약 발행
19:00   업무 종료 or 마감 압박 시 야근
20:00   퇴근 후 자기개발 (마케팅 커뮤니티 활동)

[목표]
- 클라이언트를 3개 → 6개로 늘리되 업무 시간은 늘리지 않기
- 반복 업무(SNS 카피 5종, 월간 보고서, 계약서 작성)를 자동화
- 나만의 AI 어시스턴트가 내 글쓰기 스타일을 학습해서 대신 초안을 써줬으면

[동기]
- "시간이 곧 돈"이라는 가치 방정식이 명확함
- 경쟁자 대비 납품 속도와 품질 우위를 유지하고 싶음
- 월 $30 이하면 ROI 계산 없이 바로 씀

[좌절감 (Pain Points)]
- ChatGPT에서 내 브랜드 보이스를 매번 다시 설명해야 함 → "나를 기억하는 AI가 없어요"
- 클라이언트마다 요구사항이 다른데 각자 맞는 템플릿이 없음
- AI가 생성한 콘텐츠의 품질이 들쑥날쑥 → 결국 90% 직접 수정
- 뤼튼, ChatGPT, Claude를 동시에 쓰는데 비용이 누적되고 관리가 어려움
- 한국어 비즈니스 문서(계약서, 제안서, 보고서)에 특화된 AI 도구가 없음

[사용자 인용문 (가상)]
"저는 클라이언트한테 '이 브랜드는 MZ 감성, 이건 B2B 정중한 톤'
 이렇게 머릿속에 다 있거든요. 근데 AI한테는 매번 처음부터
 설명해야 해요. 저를 기억하는 어시스턴트가 너무 필요해요."

[사용 시나리오]
시나리오: 목요일 오전, 클라이언트 3곳의 주간 콘텐츠 납품
- 지은은 매주 목요일 3개 클라이언트에게 SNS 카피 5종 + 블로그 1편 납품
- 기존에는 각 클라이언트 Notion 페이지에서 브랜드 가이드 확인 후 ChatGPT에 복붙
- AgentDesk에서 "클라이언트 A 에이전트"가 해당 브랜드 가이드, 이전 콘텐츠,
  이번 주 키워드를 토대로 초안 자동 생성
- 지은은 30% 수정만 하고 3시간 만에 납품 완료 (기존 대비 6시간 절약)
```

### Persona 3: "결재받아야 하는" 박현철 (ICP-C: 중소기업 비기술 부서장)

```
[기본 정보]
이름:     박현철
나이:     44세
직책:     마케팅 팀장
회사:     중소 제조업 B2B 기업 (직원 65명, 매출 80억)
위치:     경기도 안양
학력:     경영학 학사
팀 규모:  마케팅팀 5명 + 외부 에이전시 2개

[기술 숙련도]
툴:       Excel, 파워포인트, 카카오워크, MS Teams, 더존 ERP
AI 활용:  ChatGPT 가끔 사용 (검색 대용 수준), 회사 업무에 AI 활용한 적 없음
숙련도:   ★★☆☆☆ (디지털 툴은 쓰지만 새 도구 학습에 저항감 있음)

[일상 (하루 24시간)]
08:30   출근, 팀원들 컨디션 체크
09:00   주간 마케팅 계획 점검, 전주 성과 정리
10:00   에이전시 미팅 (광고 소재 피드백)
11:30   대표이사에게 월간 마케팅 보고서 제출
13:00   점심 (팀원 식사 + 비공식 현황 파악)
14:00   카탈로그 제작 업체 미팅
15:30   신규 유통 채널 제안서 작성
17:00   팀원 성과 면담
18:00   퇴근 전 이메일 정리 + 내일 할 일 메모

[목표]
- 팀원 5명의 업무 생산성 향상 → 에이전시 의존도 낮추기
- 대표에게 AI 도입 성과를 수치로 증명하기
- 보고서 작성, 경쟁사 분석, SNS 콘텐츠 반복 작업 시간 줄이기

[동기]
- 경쟁사가 AI 도입 후 콘텐츠 생산량이 3배 늘었다는 이야기를 들음
- 대표이사가 "우리도 AI 한번 써봐"라고 말하기 시작함
- 성공하면 팀 예산 확대와 본인 승진에 도움이 될 것이라는 기대

[좌절감 (Pain Points)]
- 영어로 된 AI 도구는 한국 B2B 마케팅 문서 작성에 맞지 않음
- 복잡한 설정이 필요한 도구는 팀원들이 쓰다가 포기함
- 데이터를 AI에 입력했을 때 정보가 유출될까봐 걱정됨 (개인정보보호법)
- 도입했다가 실패하면 "AI 반대파" 임원에게 책임 추궁 당할 수 있음
- 구매 결재를 위한 ROI 증명 자료 작성이 막막함

[사용자 인용문 (가상)]
"저는 AI를 못 믿는 게 아니에요. 근데 우리 팀에서 이거 도입했다가
 잘 안 되면 제 책임이잖아요. 쉽게 쓸 수 있고,
 한국말로 돼 있고, 보안 믿을 수 있으면 바로 결재 올릴 수 있어요."

[사용 시나리오]
시나리오: 월요일 오전, 경쟁사 분석 보고서 작성
- 현철은 분기마다 경쟁사 3곳의 마케팅 활동 분석 보고서를 대표에게 제출
- 기존에는 팀원 1명이 이틀 동안 웹 서핑, 스크린샷, 엑셀 정리 후 PPT 작성
- AgentDesk의 "경쟁사 분석 에이전트"가 지정 웹사이트·SNS를 자동 모니터링하고
  한국어 보고서 초안을 파워포인트 형태로 3시간 내 생성
- 현철은 내용 검토 1시간 후 대표에게 제출 → 팀원 이틀치 업무가 반나절로 단축
```

## 3. 핵심 사용 시나리오 5개

### 시나리오 1: AI 에이전트 생성 및 첫 실행

**사용자 스토리**
- As a: 프리랜서 콘텐츠 마케터 이지은
- I want to: 클라이언트별 브랜드 가이드를 학습한 AI 에이전트를 5분 안에 만들고 싶다.
- So that: 매번 ChatGPT에 브랜드 정보를 다시 입력하는 30분을 아낄 수 있다.

**수용 기준 (Acceptance Criteria)**
- [ ] 에이전트 생성 마법사(Wizard)는 3단계 이하로 완료 가능해야 한다.
- [ ] 텍스트 붙여넣기 또는 URL 입력만으로 브랜드 지식 베이스를 구성할 수 있어야 한다.
- [ ] 에이전트 생성 완료 후 즉시 테스트 대화가 가능해야 한다.
- [ ] 에이전트 이름, 아이콘, 설명을 사용자가 직접 커스터마이징 가능해야 한다.
- [ ] 생성 소요 시간 5분 이내 (신규 사용자 기준 사용성 테스트 통과 기준).

---

### 시나리오 2: AI 에이전트 작업 현황 모니터링

**사용자 스토리**
- As a: 스타트업 CTO 김성준
- I want to: 팀에 배포한 AI 에이전트들이 오늘 어떤 작업을 얼마나 수행했는지 한눈에 보고 싶다.
- So that: 각 에이전트의 작동 여부와 이상 징후를 빠르게 감지하고 비용을 통제할 수 있다.

**수용 기준 (Acceptance Criteria)**
- [ ] 메인 대시보드에서 에이전트별 오늘의 실행 횟수, 성공/실패율, 평균 응답 시간을 확인할 수 있어야 한다.
- [ ] 실패한 작업은 빨간색으로 표시되고, 클릭 시 오류 로그를 확인할 수 있어야 한다.
- [ ] 금일/주간/월간 LLM 비용 사용액이 실시간으로 표시되어야 한다.
- [ ] 비용 임계값을 설정하고, 초과 시 이메일 또는 슬랙 알림을 받을 수 있어야 한다.
- [ ] 에이전트 현황은 30초 이내 자동 새로고침으로 실시간성을 유지해야 한다.

---

### 시나리오 3: 에이전트 간 업무 인수인계 (핸드오프)

**사용자 스토리**
- As a: 스타트업 CTO 김성준
- I want to: "리서치 에이전트"가 수집한 정보를 자동으로 "보고서 작성 에이전트"에게 전달하고 싶다.
- So that: 에이전트 간 데이터 전달을 위해 내가 중간에서 복붙하는 수고를 없앨 수 있다.

**수용 기준 (Acceptance Criteria)**
- [ ] 에이전트 A의 출력(Output)을 에이전트 B의 입력(Input)으로 연결하는 UI가 드래그 앤 드롭으로 가능해야 한다.
- [ ] 연결된 에이전트 간 데이터 전달 형식(텍스트, JSON, 파일)을 설정할 수 있어야 한다.
- [ ] 특정 조건(키워드 포함 시, 특정 시간, 수동 승인) 기반의 핸드오프 트리거를 설정할 수 있어야 한다.
- [ ] 핸드오프 내역은 타임라인 형태로 로그에 기록되어야 한다.
- [ ] 핸드오프 실패 시 원인과 함께 사용자에게 즉시 알림이 전송되어야 한다.

---

### 시나리오 4: 산출물 버전 관리 및 비교

**사용자 스토리**
- As a: 프리랜서 콘텐츠 마케터 이지은
- I want to: 같은 에이전트가 오늘과 2주 전에 생성한 콘텐츠 초안을 나란히 비교하고 싶다.
- So that: AI 출력 품질이 개선되었는지 확인하고, 클라이언트에게 원하는 버전을 제출할 수 있다.

**수용 기준 (Acceptance Criteria)**
- [ ] 에이전트가 생성한 산출물은 자동으로 버전이 기록(타임스탬프, 프롬프트 스냅샷 포함)되어야 한다.
- [ ] 두 버전을 선택하여 차이를 하이라이트로 비교하는 Diff 뷰를 제공해야 한다.
- [ ] 이전 버전으로 1클릭 롤백(복원)이 가능해야 한다.
- [ ] 특정 버전을 "승인됨(Approved)" 상태로 표시하고, 해당 버전의 프롬프트와 설정을 저장할 수 있어야 한다.
- [ ] 버전 히스토리는 최소 90일치를 보관해야 한다 (무료 플랜은 30일).

---

### 시나리오 5: 워크플로우 템플릿 적용

**사용자 스토리**
- As a: 중소기업 마케팅 팀장 박현철
- I want to: 마케팅 업무에 바로 쓸 수 있는 AI 에이전트 구성을 1클릭으로 설치하고 싶다.
- So that: AI나 프롬프트 엔지니어링 지식 없이도 팀 전체가 즉시 사용할 수 있다.

**수용 기준 (Acceptance Criteria)**
- [ ] 직종별(마케팅, 개발, 콘텐츠, HR, 영업 등) 템플릿 카테고리가 존재해야 한다.
- [ ] 각 템플릿은 미리보기(사용 예시 스크린샷 또는 영상)를 제공해야 한다.
- [ ] 템플릿 설치 후 회사명, 업종 등 최소한의 정보만 입력하면 즉시 사용 가능해야 한다.
- [ ] 설치된 템플릿은 자유롭게 커스터마이징하거나 삭제할 수 있어야 한다.
- [ ] 사용자 커뮤니티가 공유한 템플릿을 탐색하고 1클릭 복사로 내 워크스페이스에 추가할 수 있어야 한다.

## 5. UX 원칙 5가지

AgentDesk만의 UX 디자인 원칙은 "AI를 처음 팀에 도입하는 비전문가"도,
"AI를 이미 깊이 활용하는 전문가"도 동시에 만족시키는 설계를 목표로 한다.

---

### 원칙 1: 즉시 가치 (Instant Value First)

**정의**: 사용자는 AgentDesk를 켠 지 10분 안에 유용한 결과를 경험해야 한다.

**근거**:
- 프리랜서(ICP-B)의 평균 앱 평가 시간은 10~15분 이내
- SaaS 온보딩 연구에 따르면 첫 "Aha Moment"까지의 시간(TTV)이 짧을수록 유료 전환율이 높다
- 경쟁사(Zapier, Make.com)는 초기 설정 복잡도로 인한 이탈률이 높음

**디자인 지침**:
- 가입 후 "직업 선택 → 관련 템플릿 자동 추천 → 1클릭 설치 → 즉시 실행"의 4단계 온보딩
- 빈 화면(Empty State)은 없다. 항상 샘플 에이전트 또는 가이드 행동(CTA)을 제공한다.
- 첫 작업 완료 시 절약한 시간을 수치로 표시한다 (예: "방금 약 45분을 절약했습니다").

---

### 원칙 2: 투명한 AI (Transparent AI)

**정의**: AI가 무엇을 왜 하는지 사용자가 항상 알 수 있어야 하며, 제어권은 사용자에게 있다.

**근거**:
- 중소기업 사용자(ICP-C)의 최대 불안 요소는 "AI가 잘못된 일을 해도 모른다"는 불확실성
- 에이전트 AI 설계 원칙 2026 트렌드에서 "Variable Control Level"과 "Sandbox Mode"가 핵심 패턴으로 부상
- AI 출력 품질 불일관성은 3개 ICP 모두 공통 Pain Point

**디자인 지침**:
- 에이전트의 모든 행동은 인간이 읽을 수 있는 로그로 표시한다 ("리서치 에이전트가 URL 3개를 분석했습니다").
- 중요 결정(외부로 전송, 파일 삭제)은 실행 전 반드시 사용자 승인을 요청한다.
- "신뢰도 점수" 또는 "이 응답에 사용된 소스"를 결과물과 함께 표시한다.
- 프롬프트와 설정을 사용자가 언제든 직접 볼 수 있도록 공개한다.

---

### 원칙 3: 한국인을 위한 디자인 (Designed for Koreans)

**정의**: 언어뿐 아니라 한국 비즈니스 문화와 워크플로우에 최적화된 경험을 제공한다.

**근거**:
- ICP-C(중소기업)의 핵심 불만은 "영어 중심 AI"로, 한국어 특화 경험이 차별화 포인트
- 한국 비즈니스 문서(품의서, 보고서, 공문)는 영어권 포맷과 구조가 다름
- 카카오워크, 더존 ERP 등 한국 특화 협업 도구와의 연동이 글로벌 경쟁사 대비 우위

**디자인 지침**:
- 전체 UI, 도움말, 오류 메시지, 이메일을 한국어로 제공 (영어 병기는 설정에서 선택)
- 기본 제공 템플릿에 한국 비즈니스 서식(주간 업무 보고, 품의서, 제안서) 포함
- 카카오워크, 네이버 워크플레이스 연동을 Slack보다 우선 지원
- 국내 개인정보보호법(PIPA) 준수 배지를 설정 페이지와 가격 페이지에 명시적으로 표시

---

### 원칙 4: 전문가도 만족하는 단순함 (Simple for All, Powerful for Experts)

**정의**: 비기술 사용자도 막힘 없이 쓸 수 있어야 하며, 전문가도 깊은 커스터마이징이 가능해야 한다.

**근거**:
- ICP-B(프리랜서)와 ICP-C(중소기업)는 기술 장벽에서 이탈
- ICP-A(스타트업 CTO)는 제한된 커스터마이징으로 인해 "직접 만드는" 선택을 함
- n8n과 Make.com의 사례에서, 시각적 직관성과 기술적 유연성의 균형이 사용자 만족도를 결정

**디자인 지침**:
- 기본 뷰는 "쉬운 모드"로 설정: 필수 옵션만 표시, 고급 설정은 접힌 상태
- "고급 설정 보기" 토글로 JSON 편집기, 커스텀 프롬프트, API 파라미터에 접근 가능
- 드래그 앤 드롭 워크플로우 캔버스와 YAML/JSON 코드 편집기를 탭으로 전환 가능
- 각 설정 항목에 설명 아이콘(?)을 달고, 클릭 시 예시와 설명이 나타나도록

---

### 원칙 5: 비용은 놀라움이 없어야 한다 (No Billing Surprises)

**정의**: 사용자는 언제나 현재 비용과 예상 청구액을 명확히 알 수 있어야 한다.

**근거**:
- ICP-A(스타트업 CTO)의 핵심 Pain Point: "LLM API 비용이 예측 불가 → 월말 청구서 공포"
- ICP-B(프리랜서)의 구독 결정 기준: "ROI가 불투명한 구독은 해지"
- 가격 민감도 리서치에 따르면 예측 불가능한 비용 구조는 해지의 주요 원인

**디자인 지침**:
- 대시보드 상단에 "이번 달 사용량/예산" 프로그레스 바를 항상 표시
- 새 에이전트 실행 전, 예상 비용 범위를 확인 알림으로 표시 (예: "이 작업은 약 $0.02~$0.05가 소요됩니다")
- 일간/주간/월간 비용 이메일 요약 opt-in 제공
- 무료 플랜 한도 도달 시 즉시 중단 또는 유예 선택지를 주고, 한도 80% 시점에 미리 경고
- 모든 플랜의 포함 항목과 과금 기준을 단순한 표(비교표)로 표시

## 주요 발견사항 요약

### 발견 1: "기억하는 AI"에 대한 강한 니즈

- **설명**: 사용자들은 범용 AI(ChatGPT, Claude)를 사용하지만, 매번 컨텍스트를 재입력해야 하는 반복 작업에 지쳐 있다. 자신의 업무 스타일과 클라이언트 정보를 기억하는 "맞춤형 에이전트"에 대한 니즈가 3개 ICP 모두에서 공통적으로 발견된다.
- **근거 데이터**: 시장 리서치 보고서 ICP-B/A Pain Point, 경쟁사(ChatGPT, Notion AI) 불만 사항 분석
- **영향도**: 높음
- **디자인 권장사항**: 에이전트별 지식 베이스(Knowledge Base)를 핵심 MVP 기능으로 설계하고, 온보딩 첫 단계에서 지식 베이스 설정을 반드시 거치게 한다.

### 발견 2: 한국어 비즈니스 문서 특화 기회

- **설명**: 글로벌 AI 도구들이 한국어 지원을 추가하고 있으나, 한국 비즈니스 문서 서식(품의서, 주간 보고서, 제안서)과 한국 협업 도구(카카오워크, 더존) 연동은 여전히 미충족 상태다.
- **근거 데이터**: ICP-C Pain Point("영어 중심 AI"), 한국 SaaS 시장 분석 (중소기업 AI 도입률 17%), 글로벌 경쟁사 연동 목록 분석
- **영향도**: 높음 (ICP-C에 대해), 중간 (ICP-B에 대해)
- **디자인 권장사항**: 한국 비즈니스 문서 템플릿을 런치 시점 최소 10종 이상 제공. UI 전체를 한국어 우선으로 설계하고 영어는 옵션으로.

### 발견 3: 비용 불안이 도입 장벽

- **설명**: LLM API 비용의 예측 불가능성이 스타트업 CTO의 도입 저항 요소이며, "또 다른 구독료"에 대한 부담이 프리랜서의 전환 허들이다. 투명한 비용 구조가 신뢰 형성의 핵심이다.
- **근거 데이터**: ICP-A 구매 저항 요소, ICP-B 가격 민감도 ($20 심리적 기준선), 경쟁사(LLM API 서비스) 리뷰 분석
- **영향도**: 높음
- **디자인 권장사항**: 비용 추적 대시보드를 MVP에 포함. 플랜별 포함 LLM 비용을 명시. 비용 알림 기본 설정 활성화.

### 발견 4: 온보딩 탈락이 활성화의 최대 장벽

- **설명**: 비기술 사용자(ICP-C)와 바쁜 프리랜서(ICP-B)는 초기 설정이 복잡하면 포기한다. 첫 성공 경험(Aha Moment)까지의 경로를 극단적으로 단순화해야 한다.
- **근거 데이터**: SaaS 온보딩 연구 (TTV 최소화 = 전환율 향상), Make.com/Zapier 이탈 패턴, ICP-C Pain Point("복잡해서 포기")
- **영향도**: 높음
- **디자인 권장사항**: 신규 가입자는 반드시 직업 선택 → 추천 템플릿 설치 → 첫 실행의 3단계 온보딩을 완료해야 대시보드에 진입하도록 가이드. 설정 항목은 기본값을 최적화하여 사용자 선택 요소를 최소화.

### 발견 5: 전문가(ICP-A)를 위한 깊이 있는 커스터마이징 필요

- **설명**: 스타트업 CTO는 "직접 만들 수 있다"는 자신감이 있어, 커스터마이징이 제한된 제품은 쓰지 않는다. 기술적 유연성(API 연동, 커스텀 프롬프트, LLM 선택)이 없으면 이 세그먼트는 이탈한다.
- **근거 데이터**: ICP-A 구매 저항 요소, n8n vs Make.com UX 비교 (기술 전문가는 n8n의 유연성을 선호), GitHub Star 및 문서 품질이 구매 트리거
- **영향도**: 중간 (ICP-A 세그먼트 특정)
- **디자인 권장사항**: "고급 설정" 영역에서 모델 파라미터 직접 수정, 커스텀 API 연동, 프롬프트 직접 편집 기능 제공. API 문서와 GitHub 예제를 공개하여 개발자 신뢰 확보.
